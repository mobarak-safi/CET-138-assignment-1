What is Full Stack Development?

Full stack development covers all three parts of a web app: what the user sees (frontend), the server that handles requests (backend), and the database that stores data.

Frontend (Client-Side)

What the user sees in the browser. Written in HTML, CSS and JavaScript. Frameworks like React sit in this layer.

Backend (Server-Side)

Runs on the server. Gets requests from the browser, runs the logic, talks to the database, and sends back a response. Built with Node.js, Python or PHP.

Database

Stores all the data. MySQL and PostgreSQL use structured tables with SQL. MongoDB stores data as JSON documents instead.

Three-Layer Architecture

Presentation Layer
Browser renders HTML/CSS. JavaScript runs here and sends requests to the backend.
HTML CSS JS
Application Layer
Server gets the request, runs the logic, queries the database, then returns data.
Node.js REST API
Data Layer
Stores and returns data. Queried with SQL or document-based methods.
MySQL MongoDB

How a request flows: User clicks a button → JavaScript sends a request to the server → server queries the database → server sends back JSON → JavaScript updates the page. No full reload needed.

Common Stacks

MERN Stack

MongoDB, Express.js, React, Node.js. JavaScript runs on both the frontend and backend. Useful when you want one language across the whole project.

LAMP Stack

Linux, Apache, MySQL, PHP. An older but widely used stack. Still powers a large portion of the web including WordPress sites.

HTML

HTML is the markup language used to build web pages. It uses tags to define structure, and the browser turns it into a DOM tree that JavaScript can access and change.

Elements and Tags

Tags wrap content to give it meaning. Most have an opening and closing tag. Some like <img> and <br> are self-closing.

Semantic HTML5

Tags that describe what the content is, not just how it looks. Things like <nav>, <article> and <footer> help search engines and screen readers understand the page.

CSS

CSS adds style to HTML. It controls colours, fonts, layout, spacing and how the page looks on different screen sizes.

Selectors and the Cascade

CSS targets elements using selectors. If two rules target the same element, specificity decides which one wins. IDs beat classes, classes beat tags.

Box Model

Every element is a box. It has content in the middle, then padding, then a border, then margin outside. Setting box-sizing: border-box stops elements being wider than expected.

Flexbox and Grid

Flexbox lines things up in a row or column. Grid is for two-dimensional layouts where you need control over both rows and columns at once.

Responsive Design

Media queries let you change styles at different screen widths. Mobile-first means starting with styles for small screens, then adding rules for bigger ones.

JavaScript

JavaScript runs in the browser and makes pages interactive. With Node.js it also runs on the server, so the same language works across the whole stack.

DOM Manipulation

JavaScript can read and change HTML elements in real time. You can update text, add classes, create new elements and remove them without refreshing the page.

Async / Fetch API

The Fetch API lets JavaScript call external APIs and get data back. async/await keeps the code readable. Promise.all runs several requests at the same time.

Async JavaScript and Fetch

// async / await
async function getUser(id) {
  try {
    const response = await fetch(`/api/users/${id}`);
    if (!response.ok) throw new Error(`HTTP ${response.status}`);
    const user = await response.json();
    return user;
  } catch (err) {
    console.error('Fetch failed:', err.message);
  }
}

// Run multiple requests at the same time
async function loadDashboard() {
  const [users, posts, stats] = await Promise.all([
    fetch('/api/users').then(r => r.json()),
    fetch('/api/posts').then(r => r.json()),
    fetch('/api/stats').then(r => r.json()),
  ]);
  return { users, posts, stats };
}

Form Validation

function validateForm(e) {
  e.preventDefault();

  const email = document.querySelector('#v-email').value.trim();
  const pass  = document.querySelector('#v-pass').value;
  const errors = [];

  if (!/^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email))
    errors.push('Invalid email address.');

  if (pass.length < 8)
    errors.push('Password must be 8+ characters.');

  if (!/[A-Z]/.test(pass))
    errors.push('Password needs an uppercase letter.');

  errors.length
    ? displayErrors(errors)
    : displaySuccess('Form is valid!');
}
